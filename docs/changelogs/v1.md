# v1 Changelog

## v1.0.0

### cotton-box
- This package now has the following external dependencies:
  - [`@glyph-cat/foundation`](https://www.npmjs.com/package/@glyph-cat/foundation)
  - [`@glyph-cat/type-checking`](https://www.npmjs.com/package/@glyph-cat/type-checking)

### cotton-box-react
- This package now has the following external dependencies:
  - [`@glyph-cat/foundation`](https://www.npmjs.com/package/@glyph-cat/foundation)
  - [`@glyph-cat/type-checking`](https://www.npmjs.com/package/@glyph-cat/type-checking)
- `useSimpleStateValue` and `useStateValue` now accepts `null` and `undefined` as the first parameter
  - When the state manager provided to the hooks are `null` or `undefined`, they will return `NO_STATE`
  - `NO_STATE` is a `symbol` that indicates no state manager was provided to the hooks
  - `NoState` is the type definition for `NO_STATE`
  - You can use `hasState` and `hasNoState` to check if a returned value is `NO_STATE`
  - Type Safety: if the state manager parameter is statically guaranteed to be not `null` and not `undefined`, then the return type will not include `NoState`
- Added new hook `useSimpleStateValueOnly`.
  - It is similar to `useSimpleStateValue` but does not support any additional options or selectors.
  - This hook also accepts `null` and `undefined` as the first parameter.
- Removed: `useSimpleStateValueWithReactiveSelector`, `useStateValueWithReactiveSelector`
